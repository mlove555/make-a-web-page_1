Web Basics

Quick Read

Background Information Before Jumping into Hands on Coding

The World-Wide-Web is a vast collection of documents that can be accessed from anywhere, at any time, by anyone in possession of the correct address (url). These documents, the majority of which are coded in HTML, are connected to each other by hyperlinks.

Hyper Text Markup Language (HTML) is a collection of code that defines what the users of web documents will view on web pages. Steve Huffman explains that HTML is the glue that holds the web together and hyperlinks are the means of transportation between them.

HTML Defines What the Viewer:

  • is able to see (text content)
  • what the documents (pages) will look like (markup)
  • and references (links) to other documents

Initial HTML Vocabulary Includes:

  • Tag—a letter or group of letters that are surrounded by these two brackets < > (this is an opening tag), or preceded by a forward slash in these two brackets </ > (this is a closing tag)
  • Element—the content that goes between the opening and closing tags above, this will be displayed as the tags dictate
  • Attribute—describes a property of the html element, in this case an anchor tag that provides a link. < a href=”http://www.urbandictionary.com” > the information following "href=" will provide the computer processor with the information it needs in this case to link to Urban Dictionary

Warning

Some claim that computers are stupid. Computers can’t really be stupid since they are not sentient; they operate in a binary system of electrical circuits and can only make a distinction between open or closed. A coder may also think of open or closed as yes or no.

The coder uses HTML to instruct the computer processor to create a change such as bold or italic to specific elements in the text, they must also tell the computer when to stop taking this action.

The computer’s circuits do not have the ability to ask themselves if the programmer really intended to use bold for the entire last half of the document, the programmer must anticipate this issue.

HTML is Composed of Tags Which Fall into Three Categories:

  • Mark Up—makes changes such as bold or italics
  • Anchor—code that provides directions for a web address thereby creating a link
  • Image—inserts images into web pages along with its attributes. The attributes include where the item will be found (url) along with a description or text explanation of what the item is if the link can’t be viewed. This is important for two reasons, the first is the user may be able to find the link on their own and more importantly it addresses the requirements of Universal Design, which ask the programmer to anticipate the needs of visually impaired web cruisers

Creating White Space

White Space is an essential element for making webpages easy on the viewer. When content is displayed with gaps for the eye to rest on the viewer can interact more efficiently in the non-print based medium of the World-Wide-Web.

Web pages are more likely to be read, enjoyed and revisited if the viewer is not slammed by a wall of words.

Web cruisers are not going to give a page any more attention than they need too. White space engages them by providing a quick visual means to scan for content that is interesting to them.


HTML Looks Like

Now for Some Beginning Coding

First: MarkUp --Bold and Italics

For bold the HTML code looks like: <b>I am bold</b>/em>
The viewer sees I am bold

For Italics (emphasis) the code looks like: <em> I am italic </em >
The viewer sees I am italic

Recall that the open commands are <b> and <em> and the required closing command needs a forward slash, </b> and </em>.

The processor will not stop doing any instruction you start until you tell it to stop doing so.

Second: Anchor--Providing the processor with a web address/url

The code looks like: <a href="www.linkyouwant.com">clickable name</a>
The viewer sees a clickable link clickable name


Third: Inserting an Image

The code looks like: <img src=”url” alt=”text description”>
The viewer sees the picture (jpeg that you want seen)

Notice that there is no </img> this is because inserting pictures falls into the category of void tags. These are tags that complete a task with one instruction, (insert an image or a white space) doing so without interrupting original coding structure. The alt element in this tag is important and should not be ignored as it provides the viewer with information about the image when a link is broken or the viewer cannot see it. Web designers must create alternate (hence alt=) resources to address Universal Design for Learning (UDL) principles. UDL allows for even the visually impaired to interact successfully with the world-wide-web.

Fourth: Two Methods for Creating White Space

<br> and <p>, line break and paragraph.
  • <br> is a void tag (does not need a closing tag) along with being an inline tag
  • <p>block</p> is a block tag which creates a container for larger amounts of content. It will stretch to contain anything placed within the opening and closing commands while providing a border of whitespace around the content

Fifth: Container Elements

<span> and <div>, inline and block
  • <span class=” “> text </span> is an inline container
  • <div class=” “ > text </div> is a block container

Attributes in an "Element"

id vs class

id is a unique identifier for any element on the entire page and must be used only once and it looks like

class applies a similar style to groups of elements allowing coding to be placed in the CSS file, reducing repetition.

The code looks like <div class="descriptive name">

  • Void commands do not require a stop command
  • Commands may be inline or block, spaces or containers
  • Without HTML the viewer will encounter a wall of words and jump over your site

Creating Structured Documents

Adding Structure

Begins with two core tags; span and div—neither are void tags and require opening <> and closing </>. This is a great place to start learning about the difference between these two tags.

In order to create structure for a web page the coder needs to:
  • Have some idea of what they want their content to look like.
  • Identify the layers/parts as boxes, and boxes inside of boxes.
  • Nest div tags inside of each other to reflect the structure of the boxes inside of boxes
  • Assign relevant class names in the div tag.
    This looks like <div class="relevant name" >

Adding titles in your code will create and help shape the structure of the code. The title code is only visible in the HTML code and not in the web page itself.
This looks like <title>logical title</title>

Inserting comments to describe what you are trying to accomplish enhances the readability of your code and assists other programmers when they view your code. The comment tag is similar to a void tag since it closes itself. It looks like <!---text--->, nothing in this tag will display for the viewer until they click into the developer mode. This is an excellent place for the programmer to explain their logic behind their code.

Inline, Block and Container Elements

These tags types have different effects on how the browser will display HTML code for the viewer.

Inline prompts do not instruct the browser to add line breaks or drop down to the next line.

These tags include:
  • b – bold
  • a-anchor
  • em-italic
  • img-insert an image
  • span-groups elements without creating any visual change


Block elements generally begin and end with a new line. They are also referred to as container elements and have two dimensions, width and height.

These tags include:
  • p-places text in a box with width and height
  • div-groups HTML elements


Container elements are CSS commands and boost how/where HTML displays the code it reads.

These tags are:
  • span-which is inline and begins with a code that looks like <span class=”something”>text</span>
  • div-which is a block element and begins with a code that looks like <div class=”something”>text</div>

Developing Style Using CSS

Cascading Style Sheet (CSS) is code that adds color and shapes to HTML is a uniform, succinct, efficient manner. On this web page if I want to change the color on all of the lighter blue headings I only have to change one element in the CSS.

CSS searches HTML for selectors, class names, or ids applying the assigned style information to the div containers. By coding style in the CSS repetition is avoided in HTML.

Adding Style to a Webpage

HTML creates content and controls the structure of a web page while CSS, Cascading Style Sheet, adds style. CSS reduces the amount of repetitive code required by setting a style configuration. Using these two tools creates a structure for code that makes a web page more interesting for the viewer and understandable for Front End Development sharing. The process, DOM, Document Object Model, refers to the branching structure that HTML coding looks like while cascading implies that the style will be applied to all subsequent branches unless a more specific rule is defined along the way.

Avoid Repetition

What is repetition? Repetition is occurring when you find yourself getting the great idea to copy code and paste it multiple times throughout your HTML. If you are doing this and not altering some element in the tag such as class or id then you are repeating yourself.

Why should it be avoided? Imagine that you have copied and pasted code for about the last fifty minutes in at least 73 places, now imagine that you are reviewing your work and you notice that something does not look quite right in a few places, as you dig deeper you realize that the same error is occurring in all of the places that you have just pasted copied code into. This means that you will now have to go back and find all of the places in your code where this problem exists and fix them. "ARRGGHHH", time wasted and badly.

If wasting time doesn't bother you think about a scenario in which you simply cannot figure out what is wrong with your code. You may have to ask someone for help. Now, your mom might sit down and draw lines from one tag to another while you clean her house and three days later find the answer, but do you have time for this. If your code is a mess of repetition no one else is going to look at your work.

Another problem with repetition is that if you need to make a change, or maintain your code to address updates, the action must be done in every single individual instance that it occurs within the HTML code. Another time vacuum.

How not to repeat yourself. Avoiding repetition means being alert as you code, if you find yourself coding any element or tag exactly like another try to think of a way to identify that action. Assign a class or heading to the action and describe what you want done in the CSS element. This means that if you realize you don't like something about the element you need only make the change in your CSS code one time and the computer will take care of finding the other 72 places you assigned that value.

Avoiding repetition helps anyone following your code or trying to help you with coding issues because it keeps your logic simple to understand, maintainable and exhibits industry standards for best practices.


An Effective Strategy for Creating an Organized Webpage

  1. Write all structural HTML in one file, save it with a .html extension
  2. Place all CSS code in another file, save it with a .css extension
    be sure to place both files in the same folder
  3. Add a <head> </head> element to the top of your HTML doc
  4. Place a link tag inside the head element
    link is a void tag and does not need closing
  5. Add the following attributes in the link tag
    <link rel="stylesheet" type="text/css" href="name of your css code file from step 2">
